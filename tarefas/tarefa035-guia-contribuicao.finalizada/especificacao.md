# Tarefa 035 - Guia de contribuicao

## Fase
4 - Documentacao e Exemplos

## Objetivo
Criar CONTRIBUTING.md completo que guie novos contribuidores no processo de contribuicao tanto em codigo quanto em traducoes.

## Escopo
- Criar `CONTRIBUTING.md` na raiz do repositorio
- Secao "Como Contribuir":
  - Fork e clone do repositorio
  - Setup do ambiente de desenvolvimento
  - Criar branch para feature/fix
  - Processo de commit (convencoes de mensagem)
  - Abrir Pull Request
- Secao "Contribuindo com Codigo":
  - Padroes de codigo C# (naming, formatacao, XML docs)
  - Padroes de codigo TypeScript (ESLint, Prettier)
  - Estrutura do projeto e onde adicionar novo codigo
  - Escrever testes para novas features
  - Rodar testes localmente
- Secao "Contribuindo com Traducoes":
  - Formato dos arquivos de traducao
  - Como adicionar novo idioma natural
  - Como corrigir/melhorar traducoes existentes
  - Processo de revisao de traducoes
- Secao "Reportando Bugs":
  - Template de bug report
  - Informacoes necessarias (versao, OS, logs)
  - Como reproduzir o problema
- Secao "Sugerindo Features":
  - Template de feature request
  - Processo de discussao e aprovacao
- Secao "Code Review":
  - Criterios de aprovacao
  - Tempo esperado de resposta
  - Checklist de revisao
- Secao "Codigo de Conduta":
  - Comportamento esperado
  - Comportamento inaceitavel
  - Processo de denuncia
  - Consequencias
- Criar templates de issue e PR no `.github/`:
  - `.github/ISSUE_TEMPLATE/bug_report.md`
  - `.github/ISSUE_TEMPLATE/feature_request.md`
  - `.github/PULL_REQUEST_TEMPLATE.md`
